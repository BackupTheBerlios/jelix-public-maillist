J'ai oublié de signaler deux autres nouveautés :

- ajout du modificateur jdatetime pour le moteur de template jTpl, par 
Loic. Permet donc de formater une date en indiquant le format utilisé. 
Utilise la classe jDateTime.

- ajout d'une classe, jLog, pour logger des choses dans un fichier :

    jLog::dump($variable, "message"); pour faire un dump dans une variable
    jLog::log("message") pour ecrire un simple message


Le contenu est ajouté dans un fichier situé dans var/log de 
l'application. Dans la configuration, il y a une nouvelle section

[logfiles]
default=messages.log

-> alias = fichier de log
Vous pouvez donc avoir plusieurs fichiers de log.

on peut ajouter un paramètre supplementaires à dump et log, indiquant 
l'alias du fichier dans lequel on veut stocker. Par défaut, il utilise 
l'alias "default".


Enfin pour les contributeurs : j'ai ajouté au niveau de l'arbre 
subversion des propriétés svn:ignore pour ignorer les fichiers 
temporaires et autres fichiers de conf personnalisés. Ils n'apparaissent 
donc plus lors d'un svn status ;-)
