Bonjour,


Quelques nouvelles des développements depuis la sortie de la version 
1.0pre-alpha :

Nouveautés :

- mise en place dans l'application testapp, d'un module de test 
unitaires. Il va contenir des tests unitaires pour tester les 
différentes API de Jelix. Ces tests reposent sur SimpleTest que j'ai du 
hacké pour que ça fonctionne avec PHP5 sans générer de notices/strict en 
tout genre.
- Il n'y a pour le moment que des test unitaires sur jEvent, le système 
évènementiel de Jelix. Ils m'ont d'ailleurs permis de corriger des bugs.
- développement de la prise en charge des urls significatives. Il reste 
des tests à effectuer (et donc je suis en train de metter en place des 
tests unitaires sur jUrl).
- La surcharge des templates se fait maintenant uniquement dans 
var/themes (et non plus dans var/themes/ et var/overloads/). De plus 
l'option de conf use_theme est obsolète
- Implementation de la Feature Request #1643 : on peut indiquer une 
variable dans une clé de locale dans un template (pour avoir des noms de 
clé "dynamiques" : @une.cle.$foo@
- Ajout d'un plugin de template jlocale pour pouvoir indiquer des 
paramètres en plus d'une clé.
- Implementation de la Feature Request #1638 : nouveau type de méthode 
"count" dans les daos, et ajout d'une méthode countAll générique
- ajout de jtpl::append("nom_var", $valeur)
- remplacement de jResponseHtml/Xul::add*ofBody par addContent()
- ajout d'une option de conf use_error_handler (mais c'est déstiné à un 
usage interne pour le développement de jelix


Corrections de bug :

- Correction du bug 6214 : les fichiers temp/*.list.php n'étaient pas 
dédiés à une conf
- Correction du bug 6134 : Si les propriétés de type autoincrement dans 
un record DAO ne sont pas nulles ou superieur à 1, alors elles sont 
dorénavants incluses dans la requête d'insertion lors de l'appel de la 
méthode insert($record)
- Correction du bug 5996 : Les fichiers compilés d'actions sont 
maintenant synchronisés (quand une action était supprimés, le fichier 
"compilé" correspondant ne l'était pas).
- ajout d'un exemple de profil pdo dans dbprofil.ini.php.dist, et 
corrections de bug sur le support pdo
- corrections de bugs divers et variés (merci à Loic)
