Je suis en train de lire le code de Jelix.
C'est très intéressant (surtout parce que c'est très éloigné de mon
propre php-fu-style ;)).

Je vais essayer d'envoyer de petits patchs quand j'ai un peu de temps
(j'en ai posté un minuscule tout à l'heure :p).

Sinon, le truc le plus "familier" pour moi là-dedans, c'est http.
Question authentification, j'ai déjà implémenté digest en php. Ca
pourrait peut-être resservir.
Côté session, on peut facilement réécrire complètement le méchanisme
de session de php.
Les avantages sont nombreux:
1 - on peut utiliser la nonce digest pour trimbaler le token de
session (on a alors une session cookie-less / url-less)
2 - au passage c'est plus direct pour choisir la back-end pour le
stockage des données
3 - ça évite de se prendre la tête avec les problèmes liés à la conf
session de php (/tmp accessible en lecture, conf du cookie, etc,
etc...)


Toujours question http, pourquoi ne pas:
- gzipper les contenus à la volée (après avoir vérifié que le client
peut lire, et que le serveur peut compresser)
- implémenter un système de cache cohérent (mais je n'ai pas encore
assez lu ton code, c'est peut-être déjà fait)

D'autre part, je ne vois rien (après lecture rapide) dans ton code
pour prendre en compte les particularités de configuration de php.
Une petite couche de gestion ne serait peut-être pas inutile?
