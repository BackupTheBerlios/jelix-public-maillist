Justement, comme c'est une bizarrerie, on le met dans un plugin. Ainsi, 
pour une plateforme correctement configuré (pas de magicquotes), il n'y 
a pas à activer le plugin, donc c'est du code en moins dans le framework 
à parser pour rien. Pour celui qui est sur un serveur avec les 
magicquotes activés, il faut qu'il active le plugin :-)


> 
> Personnellement j'aurai tendance à inclure systématiquement la détection.

comme j'écrivais juste avant, une plateforme bien configurée, et une 
plateforme qui n'active pas les magicquotes. Inclure donc la detection 
ne sert à rien la plupart du temps sinon à inclure du code inutile. Et 
cela va être d'autant plus vrai à l'avenir que les magicquotes vont être 
abandonnée.
